# Qualitative Research Data Crawler -- Technical Plan (Updated)

## Project Goal

Build a Python pipeline that discovers, collects, and archives
**qualitative research projects** from several repositories.\
The crawler must identify datasets containing **qualitative data or QDA
analysis files**, download accessible project files, and store metadata
in an **SQLite database**.

Target repositories:

-   UK Data Service
-   Databrary
-   Zenodo
-   AUSSDA (Dataverse instance)

The system should prioritize datasets that contain **QDA analysis
files** but should also collect projects described as qualitative
research even if no QDA file is present.

------------------------------------------------------------------------

# 1. Definition of a Qualitative Research Project

A project is treated as a **container** (dataset/study/volume) that may
include:

### Analysis Data Files (QDA files)

Files generated by qualitative analysis software.

### Primary Data Files

Examples: - interview transcripts - field notes - research articles -
audio/video interviews

### Additional Data

Examples: - codebooks - readme files - research documentation -
methodology notes

The crawler should download **all available files in the project
container**.

# 6. Metadata Database (Assignment Requirement)

The metadata must be stored in a **SQLite database using one table**,
where **each row represents one downloaded QDA dataset**.

## Table: datasets

  -------------------------------------------------------------------------------
  Field                 Type          Required      Description
  --------------------- ------------- ------------- -----------------------------
  id                    INTEGER       yes           Primary key

  qda_file_url          TEXT          yes           URL of the QDA file

  download_timestamp    TEXT          yes           Timestamp of the most recent
                                                    download

  local_directory       TEXT          yes           Name of the local folder
                                                    containing the dataset files

  local_filename        TEXT          yes           Name of the downloaded QDA
                                                    file

  repository_name       TEXT          no            Name of the repository

  dataset_page_url      TEXT          no            URL of the dataset landing
                                                    page

  dataset_title         TEXT          no            Title of the dataset

  dataset_description   TEXT          no            Dataset description

  dataset_doi           TEXT          no            DOI identifier if available

  keywords              TEXT          no            Dataset keywords

  download_method       TEXT          no            How dataset was acquired

  download_status       TEXT          no            success / partial / failed

  file_count            INTEGER       no            Number of files downloaded

  uploader_name         TEXT          no            Dataset uploader

  uploader_email        TEXT          no            Email of uploader

  author_name           TEXT          no            Dataset author

  contact_person        TEXT          no            Contact person

  license               TEXT          no            Dataset license
  -------------------------------------------------------------------------------

------------------------------------------------------------------------

## SQLite Table Definition

``` sql
CREATE TABLE datasets (
    id INTEGER PRIMARY KEY,
    qda_file_url TEXT NOT NULL,
    download_timestamp TEXT NOT NULL,
    local_directory TEXT NOT NULL,
    local_filename TEXT NOT NULL,
    repository_name TEXT,
    dataset_page_url TEXT,
    dataset_title TEXT,
    dataset_description TEXT,
    dataset_doi TEXT,
    keywords TEXT,
    download_method TEXT,
    download_status TEXT,
    file_count INTEGER,
    uploader_name TEXT,
    uploader_email TEXT,
    author_name TEXT,
    contact_person TEXT,
    license TEXT
);
```

# 8. Download Rules

For each qualifying dataset:

Download:

-   QDA files
-   documentation files
-   associated primary data files

Do NOT download:

-   files requiring authentication
-   dynamically generated pages

For restricted datasets:

    download_status = restricted

Metadata must still be stored.

------------------------------------------------------------------------

# 9. Deduplication

Avoid duplicates using:

1.  repository dataset id
2.  DOI
3.  normalized title comparison

Files should be deduplicated using SHA256 hash.
